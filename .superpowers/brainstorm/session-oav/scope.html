OpenAgentVisualizer — Where should we start?

All 14 planning documents are complete. Time to write code. Choose the starting scope for development.

A

MVP Core — Web App Only RECOMMENDED

Build the full web MVP in one focused effort: FastAPI backend + PostgreSQL/Redis/TimescaleDB + React+PixiJS virtual world + Python SDK. This is the 8-week roadmap described in the README — virtual world, agent avatars, real-time status, XP/leveling, basic metrics, session replay.

Pros

  • Ship something real and usable
  • Web-first = broadest reach
  • All planning docs cover this completely

Cons

  • Largest scope upfront
B

Backend First — API + Ingestion Core

Build the FastAPI backend, WebSocket server, OTLP ingestion, PostgreSQL/Redis schema, and Python SDK first — before any frontend. Gives a working data pipeline that can be validated with curl/Postman before the visual layer is built.

Pros

  • De-risks data pipeline early
  • SDK usable immediately for agent instrumentation

Cons

  • No visual output until frontend done
  • Harder to demo
C

Vertical Slice — One Agent, End to End

Build the thinnest possible slice that works end-to-end: one Python SDK call → backend ingest → WebSocket push → single animated agent avatar in the browser. No gamification, no dashboards — just the core loop proven from SDK to screen.

Pros

  • Fastest path to a working demo
  • Validates the whole architecture at once

Cons

  • Not shippable on its own
  • Requires subsequent phases to fill in features
